# Grimoireチュートリアル

## はじめに

Grimoireへようこそ！このチュートリアルでは、魔法陣を描いてプログラムを作成する方法を学びます。

## 最初のプログラム: Hello World

### 1. 基本的な出力

最も簡単なGrimoireプログラムは以下のような魔法陣です：

```
    ◎
    |
    ☆
    |
"Hello World"
```

- `◎`: メインエントリーポイント（二重円）
- `☆`: 出力操作（5点星）
- `"Hello World"`: 出力する文字列

### 2. 画像として描く

1. 白い紙またはデジタルキャンバスを用意
2. 黒いペンまたはブラシを使用
3. はっきりとした線で図形を描く
4. 図形間の接続線を忘れずに

## 変数と計算

### 変数の宣言

```
    ◎
    |
    □ x = 42
    |
    □ y = 13
    |
    □ sum = x + y
    |
    ☆ sum
```

- `□`: 変数を表す四角形
- 変数名は図形の中または近くに書く
- `=`: 代入を表す

### 算術演算

```
    □ a = 10
      \
       (+) ← 加算シンボル
      /
    □ b = 5
      |
    □ result
```

## 条件分岐

### if文の実装

```
        △ (x > 10)
       / \
      /   \
     ☆     ☆
   "大きい" "小さい"
```

- `△`: 条件判定の三角形
- 左側: 条件がtrueの場合
- 右側: 条件がfalseの場合

## ループ

### 基本的なループ

```
    ⬟ (5)
    |
    ☆ "Hello"
```

- `⬟`: 五角形はループを表す
- `(5)`: ループ回数
- ループ本体は五角形に接続

### 無限ループ

```
    ⬟ (∞)
    |
    △ (終了条件)
    |
    ● break
```

## 関数

### 関数定義

```
○ add(a, b)
├─ □ result = a + b
└─ ☆ result

◎
├─ □ x = add(3, 4)
└─ ☆ x
```

- `○`: 通常の円は関数を表す
- パラメータは円の近くに書く
- 関数本体は円に接続

## 配列とコレクション

### 配列の作成

```
□-□-□ array = [1, 2, 3]
  |
  ☆ array[1]
```

- 連結された四角は配列を表す
- インデックスアクセスは`[n]`で表記

## エラー処理

### try-catch

```
    ○ (try)
    |
  ~~~~ risky_operation()
    |
    ○! (catch)
    |
    ☆ "エラー発生"
```

- `~~~~`: 波線は例外を投げる可能性がある操作
- `○!`: 感嘆符付きの円はcatchブロック

## 並列処理

### 並列実行

```
      ⬢
     /|\
    / | \
   ○  ○  ○
  f1 f2 f3
    \ | /
     \|/
      ⬢
```

- `⬢`: 六角形は並列実行を表す
- 上の六角形: タスクを分散
- 下の六角形: 結果を収集

## 実践的な例

### フィボナッチ数列

```
○ fib(n)
├─ △ (n <= 1)
│  ├─ ☆ n
│  └─ □ a = fib(n-1)
│     □ b = fib(n-2)
│     ☆ a + b

◎
├─ □ result = fib(10)
└─ ☆ result
```

### FizzBuzz

```
◎
└─ ⬟ (i = 1..100)
    └─ △ (i % 15 == 0)
        ├─ ☆ "FizzBuzz"
        └─ △ (i % 3 == 0)
            ├─ ☆ "Fizz"
            └─ △ (i % 5 == 0)
                ├─ ☆ "Buzz"
                └─ ☆ i
```

## 描画のコツ

### 1. 明確な図形
- 図形の境界をはっきりと描く
- 円は閉じているか確認
- 角がある図形は角をしっかり

### 2. 接続線
- 線は図形の境界に確実に接触
- 交差する線は明確に交差を示す
- フローの方向を矢印で示すことも可能

### 3. テキスト
- 読みやすい文字で書く
- 図形内に収まるようにサイズ調整
- 必要に応じて図形の外に注釈

### 4. 空間の使い方
- 関連する要素は近くに配置
- 複雑なプログラムは領域を分けて整理
- 余白を適切に使う

## デバッグ

### デバッグシンボル

```
○ complex_function()
├─ 👁 x  ← 変数をウォッチ
├─ 🐛    ← ブレークポイント
└─ 📍 assert(x > 0)  ← アサーション
```

## まとめ

Grimoireでは、プログラムの構造が視覚的に表現されます。これにより：
- プログラムフローが一目で理解できる
- 並列処理やスコープが直感的
- デバッグが視覚的に行える

さあ、あなたも魔法陣を描いてプログラミングを始めましょう！
